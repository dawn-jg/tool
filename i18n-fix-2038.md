# 国际化(i18n)修复

## 问题
用户反映切换语言后页面文本不更新。

## 根因
`page.tsx` 是服务器组件，直接硬编码了中文文本，没有使用 `useI18n()` hook。而 `I18nProvider` 只包裹了 `children`，服务器组件拿不到 context。

## 解决方案
1. `page.tsx` 改为 `"use client"` 并使用 `useI18n()` hook 动态翻译
2. `home.toolsCount` 改为 `" 个工具"`（前面加空格），配合 `{cat.count}{t("home.toolsCount")}` 实现"10 个工具"
3. 添加了 `tools.jsonFormatter`、`tools.base64` 等热门工具的翻译 key

## 文件变更
- `app/page.tsx`: 服务器组件 → 客户端组件，使用 `t()` 翻译
- `lib/i18n.tsx`: 添加 `home.toolsCount` 空格，添加工具翻译 key
- `components/LanguageSwitcher.tsx`: 添加 console.log 调试

## 验证
控制台确认 `onChange` 触发 → 调试成功 → 问题在于 page.tsx 没用 i18n
